# Claude Pet 🪄

Claude Code가 작업하는 동안 화면 구석에서 상태에 따라 움직이는 데스크톱 캐릭터 펫입니다.
Claude Code의 **훅(hooks)** 이 현재 상태를 파일에 기록하면, 펫이 그 파일을 읽어 모션을 바꿉니다.

---

## 크기

캐릭터는 작게(초기 버전의 절반) 표시됩니다.

| 항목 | 값 | 위치 |
|---|---|---|
| 창 크기 `WIN_W` × `WIN_H` | 100 × 150 | `main.js` |
| 캐릭터 `.pet` | 75 × 105 | `index.html` |
| `.char-wrap` | 64 × 98 | `index.html` |

> 창 높이(150) 중 캐릭터 위쪽 여백은 **말풍선이 머리 위에 뜨도록** 비워 둔 공간입니다.
> 창의 화면상 위치는 `WIN_W/WIN_H` 기준으로 자동 계산되므로, 창 높이를 바꿔도 캐릭터의 화면상 위치·크기는 그대로 유지됩니다.

## 상태 ↔ Claude Code 이벤트

| 펫 상태 | 모션 / 소품 | Claude Code 이벤트 |
|---|---|---|
| `idle` 대기 | 가벼운 숨쉬기 🙂 | 세션 시작(SessionStart) |
| `thinking` 생각 중 | 고개 갸웃 💭 | 프롬프트 입력(UserPromptSubmit) |
| `working` 작업 중 | **마법봉 위/아래 프레임을 천천히 교차** 🪄 | 도구 실행(Pre/PostToolUse) |
| `waiting` 입력 대기 | 두리번 ❓ | 알림(Notification) |
| `resting` 쉬는 중 | 좌우로 살랑 ☕ | 응답 완료(Stop) |
| `sleeping` 잠 | Zzz 💤 | 세션 종료(SessionEnd) |

> 훅은 사용자 전역 설정 `~/.claude/settings.json` 에 등록되어 있습니다.
> 기존에 있던 `.pixel-agents` 훅은 그대로 보존하고 펫 훅을 **함께** 추가했습니다.

### "작업 중" 마법봉 애니메이션

- `wand_up.png`(마법봉 위) ↔ `wand_down.png`(마법봉 아래) 두 프레임을 **2초 주기(1초마다 교체)** 로 교차해, 마법봉을 천천히 흔드는 모션을 만듭니다.
- 여기에 더해 몸이 좌우로 아주 살짝 흔들리는(±1px, ±1.2°) 잔잔한 모션이 함께 적용됩니다.
- 교차 속도를 바꾸려면 `index.html` 의 `wandUp` / `wandDown` 애니메이션 시간(`2s`)을 조정하세요.

> 마법봉 두 이미지는 **발 중심 기준으로 정렬**해 동일한 캔버스 크기로 맞춘 가공본입니다.
> (정렬하지 않으면 프레임 교차 시 몸통이 좌우로 튑니다.) 원본은 `마법봉up.png` / `마법봉down.png`.

## 실행

```bash
cd 팻앱
npm install   # 최초 1회 (Electron 설치)
npm start
```
또는 `start.bat` 더블클릭.

펫은 기본적으로 **화면 오른쪽 아래 구석**에 나타납니다.

> 코드(`main.js`, `index.html` 등)를 수정하면 **앱을 재시작해야** 반영됩니다.
> Electron은 시작할 때 코드를 한 번만 읽기 때문입니다.

## 조작

- **이동**: 캐릭터를 마우스로 끌기
- **숨기기/보이기·종료**: 작업표시줄 오른쪽 **시스템 트레이의 펫 아이콘** 클릭 / 우클릭 메뉴
- 캐릭터에 마우스를 올리면 우상단에 작은 **× (종료)** 버튼이 나타남

## 부팅 시 자동 실행

1. `Win + R` → `shell:startup` 입력 → 시작프로그램 폴더 열림
2. `start-hidden.vbs` 의 **바로가기**를 그 폴더에 복사
   - 콘솔 창 없이 조용히 펫만 실행됩니다.

## 미리보기 (코드 수정 시)

`index.html` 을 브라우저에서 바로 열면 `petAPI` 가 없으므로 **데모 모드**로 동작해
6가지 상태를 2.8초마다 자동 순환합니다. 애니메이션 확인용으로 사용하세요.

## 문제 해결

- **펫이 안 보임**: 화면 맨 오른쪽 아래 확인 / 트레이 아이콘 클릭해 보이기.
- **변경이 반영 안 됨**: 앱을 **종료 후 재시작**하세요. 실행 중인 앱은 수정 전 코드를 메모리에 올려둔 상태입니다.
- **상태가 안 바뀜**: Claude Code를 재시작하거나 `/hooks` 메뉴를 한 번 열어 설정을 다시 읽게 하세요. `pet-status.json` 의 내용이 바뀌는지 확인하면 훅 동작 여부를 알 수 있습니다.
- **수동 상태 변경(테스트)**: `node write-status.js working` 처럼 직접 실행해 보세요.

## 이미지 가공 메모

- 흰 배경 제거: `python tools/cutout_white.py <원본> <출력.png> [미리보기.png] [임계값]`
- 마법봉 프레임 정렬: 두 이미지의 불투명 영역을 잘라낸 뒤, 바닥 8% 영역(발)의 가로 중심을 맞추고 동일 캔버스(바닥 정렬)로 합쳐 `assets/wand_up.png` / `assets/wand_down.png` 로 저장했습니다.
